Opdracht 1: Project organisatie¶

Voorbereiding¶

  • Lees voor het maken van de opdracht het eerste hoofdstuk: Project organisatie

  • Lees over de casus: Introductie casus

  • Download de voorbeelddata hier als je die nog niet hebt.

Opdracht ¶

We zijn Nick erg dankbaar dat we mee mogen werken aan het project, maar er moet nog wel het een en ander gebeuren. Aan ons de taak om het project beter te organiseren zodat we verder kunnen met de data:

  1. Maak een projectfolder voor het gehele project.

  2. Unzip de voorbeelddata en zet alle losse bestanden in de projectfolder.

  3. Neem kort de tijd om de bestanden te inventariseren/inspecteren:

    • Waar kijk je eerst?

    • Wat zijn de extensies van de bestanden?

    • Kan je de bestanden openen?

  4. Maak een logische structuur aan voor de bestanden (voorbeeld):

    • Waar moet de data?

    • Waar moeten de documenten?

    • Waar moeten de uitkomsten?

    • Waar moeten de scripts?

  5. Pas de bestandsnamen aan:

    • Zijn alle namen descriptief?

    • Hoe zit het met de versienummering?

    • Worden er geen spaties of speciale tekens gebruikt?

  6. Pas de README aan voor de gekozen mapstructuur:

    • Open de README met een text editor (bijv. Notepad, TextEdit, of Gedit).

    • Vul de README aan, gebruik eventueel markdown syntax.

    • Tip: Gebruik een markdown editor zoals Typora of Joplin. Fontys Research Drive bevat ook een markdown editor!

    • Wat zou nog meer in de README kunnen?
